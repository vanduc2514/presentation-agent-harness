# Impress.js Prezi-Style Presentations with Design Patterns

Non-linear storytelling presentations using impress.js data attributes and structural patterns.

## When to Apply

- Building non-linear storytelling presentations with spatial navigation
- Creating complex slide hierarchies with conditional paths
- Implementing custom animation sequences between presentation segments

## Critical Rules

**Slide State Management**: Use CSS classes generated by impress.js for animations

```css
/* WRONG - animating slides directly */
.step {
  transform: rotate(360deg);
  transition: transform 2s;
}

/* RIGHT - using impress.js state classes */
.present .rotating {
  transform: rotate(360deg);
  transition: transform 2s ease-in-out;
  transition-delay: 0.5s;
}

.future {
  opacity: 0;
  pointer-events: none;
}
```

**Spatial Positioning**: Combine multiple data attributes for complex layouts

```html
<!-- WRONG - only using x,y positioning -->
<div class="step" data-x="1000" data-y="500">

<!-- RIGHT - spatial space with rotation and scale -->
<div class="step" 
     data-x="2000" 
     data-y="1000" 
     data-z="-1500"
     data-rotate-x="30" 
     data-rotate-y="45"
     data-scale="2">
```

## Key Patterns

### Composite Pattern - Hierarchical Slide Organization

```javascript
class PresentationSlide {
  constructor(id, content) {
    this.id = id;
    this.content = content;
    this.children = [];
  }

  add(slide) {
    this.children.push(slide);
    slide.parent = this;
  }

  render() {
    return `
      <div id="${this.id}" class="step" ${this.getDataAttributes()}>
        ${this.content}
      </div>
    `;
  }
}

class SlideSection extends PresentationSlide {
  render() {
    let html = super.render();
    this.children.forEach(child => {
      html += child.render();
    });
    return html;
  }
}

// Usage
const intro = new PresentationSlide("intro", "<h1>Welcome</h1>");
const features = new SlideSection("features", "<h1>Features</h1>");
features.add(new PresentationSlide("feature1", "<h2>Fast</h2>"));
features.add(new PresentationSlide("feature2", "<h2>Secure</h2>"));
```

### Decorator Pattern - Animation Enhancements

```javascript
class SlideDecorator {
  constructor(slide) {
    this.slide = slide;
  }

  render() {
    return this.slide.render();
  }
}

class AnimatedSlideDecorator extends SlideDecorator {
  constructor(slide, animationClass) {
    super(slide);
    this.animationClass = animationClass;
  }

  render() {
    const slideHtml = this.slide.render();
    return slideHtml.replace('class="step"', 
      `class="step ${this.animationClass}"`);
  }
}

class RotatingSlideDecorator extends SlideDecorator {
  render() {
    return this.slide.render().replace(
      '</div>',
      '<div class="rotating">' + this.slide.content + '</div></div>'
    );
  }
}

// Usage
let slide = new PresentationSlide("demo", "<h1>Demo</h1>");
slide = new AnimatedSlideDecorator(slide, "fade-in");
slide = new RotatingSlideDecorator(slide);
```

### Facade Pattern - Presentation Controller

```javascript
class PresentationFacade {
  constructor() {
    this.api = impress();
    this.slides = new Map();
    this.navigationPaths = new Map();
  }

  initialize() {
    this.api.init();
    this.setupCustomNavigation();
  }

  createNonLinearPath(fromSlide, keyBindings) {
    this.navigationPaths.set(fromSlide, keyBindings);
  }

  setupCustomNavigation() {
    document.addEventListener('keydown', (e) => {
      const currentSlide = document.querySelector('.step.present');
      if (!currentSlide) return;

      const paths = this.navigationPaths.get(currentSlide.id);
      if (paths && paths[e.key]) {
        this.api.goto(paths[e.key]);
      }
    });
  }

  addStoryPath(slides, positions) {
    slides.forEach((slide, index) => {
      const position = positions[index];
      document.getElementById(slide).setAttribute('data-x', position.x);
      document.getElementById(slide).setAttribute('data-y', position.y);
      document.getElementById(slide).setAttribute('data-z', position.z || 0);
    });
  }
}

// Usage
const presentation = new PresentationFacade();
presentation.createNonLinearPath('overview', {
  '1': 'intro',
  '2': 'features', 
  '3': 'conclusion'
});
presentation.addStoryPath(
  ['intro', 'problem', 'solution'], 
  [{x: 0, y: 0}, {x: 1500, y: -800}, {x: 3000, y: 0}]
);
```

### Bridge Pattern - Animation Engine Abstraction

```javascript
class AnimationEngine {
  animate(element, properties) {
    throw new Error("Must implement animate method");
  }
}

class CSSAnimationEngine extends AnimationEngine {
  animate(element, properties) {
    Object.assign(element.style, properties);
  }
}

class WebAnimationEngine extends AnimationEngine {
  animate(element, properties) {
    element.animate([properties], { duration: 1000 });
  }
}

class SlideTransition {
  constructor(animationEngine) {
    this.animationEngine = animationEngine;
  }

  executeTransition(slide, effect) {
    const element = document.getElementById(slide);
    this.animationEngine.animate(element, effect);
  }
}
```

```html
<div id="impress" data-transition-duration="1500">

```

## Common Mistakes

- **Linear thinking** — Don't position slides sequentially. Use spacial creatively
- **Missing state classes** — Always use `.present`, `.future`, `.past` for animations
- **Hardcoded navigation** — Implement conditional paths with goto() API instead of fixed sequences
- **Overcomplicated hierarchies** — Keep Composite pattern simple; avoid deep nesting beyond 3 levels
- **NO GPU RENDERING** - Do not use any css class that involve in expensive cost of using gpu render
